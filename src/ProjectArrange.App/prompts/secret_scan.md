目标：识别项目中可能属于敏感信息或密钥泄漏的内容，并输出可操作的修复建议。

输入：
- 项目结构与关键文件内容（只提供必要片段）
- git diff（staged/unstaged）
- 最近 N 次提交的变更摘要或命中片段
- 现有扫描器（如 gitleaks）输出

输出（必须结构化）：
1) 命中列表：文件路径 + 行号范围 + 命中类型（token/secret/key/password/endpoint 等）+ 置信度
2) 风险说明：为何判断为敏感、潜在影响范围
3) 处置建议：
   - 立即措施（撤销/轮换/吊销）
   - 代码处置（移除、改用环境变量、.gitignore、git filter-repo/ BFG）
   - 预防措施（pre-commit、CI 扫描、最小权限）
4) 可执行清单：按优先级列出下一步命令与操作（不包含任何真实密钥）

硬约束：
- 不输出、不复述任何真实密钥；对疑似密钥一律用 [REDACTED] 代替
- 任何建议命令都要先提示“在确认无误后执行”
